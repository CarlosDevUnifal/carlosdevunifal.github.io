Bem-vindo ao nosso tutorial sobre Git e GitHub

Git é um sistema de controle de versões distribuído, enquanto o GitHub é uma plataforma de hospedagem de código-fonte com controle de versão usando o Git.

Comandos Git

Aqui estão alguns dos comandos Git mais usados:

Comando Descrição
git init Inicia um novo repositório Git.
git clone Obtém uma cópia de um repositório Git existente.
git branch Cria, lista, renomeia e deleta branches.
git checkout Muda para um branch ou restaura arquivos do diretório de trabalho.
git status Mostra o estado do diretório de trabalho.
git diff Mostra as mudanças entre commits, commit e working tree, etc.
git add Adiciona conteúdo ao index.
git commit Grava as mudanças no repositório.
git push Atualiza referências remotas juntamente com objetos associados.
git pull Busca e integra com outro repositório ou um branch local.
git revert Desfaz commits existentes.
git merge Une dois ou mais históricos de desenvolvimento juntos.
git stash Guarda e restaura modificações não finalizadas.
git rm Remove arquivos do índice de trabalho e do diretório de trabalho.
git config Obtém e define variáveis de configuração do Git.
git reset Reseta o HEAD atual para um estado especificado.
git remote Gerencia o conjunto de repositórios "rastreados".
git fetch Busca objetos e referências de outro repositório.
git show Mostra vários tipos de objetos.
git help Mostra a página de ajuda dos comandos Git.
git rebase Reaplica commits em cima de outro branch.
git blame Mostra quem fez cada alteração em um arquivo.
git tag Cria, lista, deleta ou verifica uma tag assinada com GPG.

GitHub

O GitHub é uma plataforma incrível para hospedagem de código-fonte. Ele permite que os desenvolvedores colaborem em projetos, independentemente de onde estejam no mundo.
